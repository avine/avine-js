Espace de noms

AvineJS ajoute à l'espace de noms Javascript l'entrée avine (ou window.avine). Les différentes fonctionnalités du framework sont disponibles en temps que propriétés de l'objet avine :

  • tool est un objet offrant un ensemble de fonctions utilitaires.
  • dom est un objet offrant un ensemble de fonctions dédiées à la manipulation le DOM.
  • Core est une classe offrant une syntaxe procédurale pour réaliser des chaînes de traitements asynchrones.
  • $ est une fonction qui combine dom et Core et permet de manipuler le DOM en travaillant par collections d'éléments.

	
	




La classe Core implémente l'héritage et est donc extensible.
	Elle possède une classe enfant : Core.$.
	La fonction $() est en quelque sorte une enveloppe qui retourne simplement une instance new Core.$().



La fonction $ est une fonction de haut niveau, dédiée à la manipulation du DOM.
	A ce titre, elle possède deux alias dans l'espace de noms principal : $ (ou window.$) et avn.



Pour simplifier l'écriture du code, la fonction $ possède également un alias de chaque fonctionnalité de avine.
	Par exemple, $.tool est un alias de avine.tool.
	Ainsi, vous pouvez écrire tout le code AvinsJS en utilisant l'unique variable $.




	

	
	




Mais, la variable $ est souvent utilisée par les frameworks et peut donc faire l'objet de collisions dans l'espace de noms.
	C'est pourquoi AvineJS possède en interne, une copie de la valeur (éventuelle) de cette variable avant de l'avoir affecté.
	Pour restaurer $ à cette valeur, il suffit d'exécuter le code avine.$.restore('$').



Pour écrire du code entièrement portable, il faut toujours passer par l'espace de noms dédié avine.
	Par exemple, la fonction $ est disponible dans avine.$.
	Vous pouvez également importer dans l'espace de nom Javascript d'autres alias de la fonction $, avec la fonction avine.$.expose.

Premiers pas...


Pour utiliser AvineJS, il faut inclure le fichier avine-js.js dans vos pages Web.
	Pour profiter des fonctions de "easing" dans les animations, il faut également ajouter le fichier easing.js.



<!-- Required -->
<script src="http://js.avine.fr/avine-js.js"></script>
<!-- Optionnal -->
<script src="http://js.avine.fr/easing.js"></script>








Conventions


Dans cette documentation, l'espace de noms avine est sous-entendu.
	Par exemple, la fonction avine.tool.cleanString est simplement nommée tool.cleanString.
